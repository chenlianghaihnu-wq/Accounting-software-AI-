# 轻账软件说明书

版本：1.0.0  
开发者：CHENLH  
适用平台：Android 8.0 及以上  
定位：本地优先的个人智能记账工具

## 一、软件简介

轻账是一款面向个人日常使用的记账软件。它的核心目标是让你用尽量少的操作完成记账：可以输入一句自然语言，例如“今天中午吃了牛肉面23”，也可以手动填写金额、大类、小类和备注；可以拍小票识别，也可以导入 CSV、JSON、XLSX、图片或整段文字。

软件的数据默认保存在手机本机数据库中。未主动配置云端模型或 OCR 时，普通记账、流水查看、统计、预算、规则管理等功能都在本机完成。涉及云端文字模型、小票整理模型、百度 OCR 等功能时，程序会把你主动提交的文本或图片内容发送到你配置的服务。

## 二、安装与首次使用

### 1. 安装 APK

把 `app-release.apk` 发送到手机后，在手机文件管理器中点击安装。如果系统提示“禁止安装未知来源应用”，需要在系统设置中允许当前文件管理器或浏览器安装未知来源应用。

### 2. 关于覆盖安装

Android 要求同一个应用包名升级时签名必须一致。如果你之前安装的是未签名测试版或 debug 版，正式签名版可能无法直接覆盖安装。遇到这种情况需要先卸载旧版再安装新版。卸载会清除该手机上的本地账单数据，请先导出或备份。

如果之前安装的是同一套 CHENLH 签名的版本，新版可以覆盖安装，并保留旧数据库。

### 3. 首次声明

第一次打开软件时会显示开发者声明。你需要点击“同意并使用”才能进入软件。如果点击“不同意，退出”，程序会直接退出。

声明内容可以在“我的”页面右上角 `⋮` 中再次查看。

## 三、首页

首页是最常用的记账入口，包含快速记账、手动记账、小票识别和最近账单展示。

### 1. 一句话记账

在首页输入框输入账单文字，然后点击“识别并保存账单”。

示例：

```text
今天中午吃了牛肉面23
昨天打车18.5
工资到账5000
奶茶12，公交2，晚饭36
```

程序会自动识别金额、类型、大类、小类、消费项和时间，并保存到账单数据库。

如果一段文字里包含多个金额，程序会尽量拆成多条账单。例如：

```text
早餐8，公交2，午饭23
```

通常会生成三条记录。

### 2. 空输入时手动记账

如果首页输入框为空，点击“识别并保存账单”不会再提示错误，而是打开“手动记账”窗口。

手动记账可填写：

- 类型：支出、收入、转账。
- 金额：必填，例如 `23`、`23.5`。
- 大类：例如餐饮、交通、购物、收入、其他。
- 小类：可选，来自当前大类下已有小类。
- 消费项 / 事项：例如牛肉面、公交、工资。
- 备注：可选。
- 标签：可选，用逗号、顿号分隔。
- 时间：可选，支持 `2026-05-02`、`2026-05-02 18:30`、`2026/05/02 18:30` 等格式。

金额是必填项。时间不填时，默认使用当前时间。

手动记账里的大类和小类采用多行点选方式。选择支出、收入或转账后，大类区域会直接显示可用大类；点击某个大类后，小类区域会刷新为该大类下的小类，继续点击即可选中。

### 3. 小票照片识别入账

点击“小票照片识别入账”，可以选择拍照或从相册选择图片。

识别流程是：

1. 图片先交给百度 OCR 识别成纯文本。
2. OCR 文本再交给“小票整理模型”。
3. 小票整理模型按商品或服务明细生成多条账单。
4. 程序保存生成的账单。

使用此功能前，需要在“模型配置”里填写百度 OCR 的 API Key、Secret Key，以及小票整理模型的 API Key、Base URL 和模型名称。

### 4. 首页账单列表

首页会按日期展示最近账单。每个日期标题会显示类似“4月28日 周二”，旁边以浅色小字显示年份。

账单行左侧显示大类色块，右侧显示金额。中间显示消费项、小类、大类。显示规则大致是：

- 有消费项和小类：消费项 · 小类 · 大类。
- 有消费项但无小类：消费项 · 大类。
- 只有小类：小类 · 大类。
- 只有大类：大类。

如果账单有退款，会在总金额下方以较小字体显示退款金额。实际统计金额按“总金额 - 退款金额”计算。

### 5. 编辑、退款与删除

点击一条账单可以进入编辑窗口。

可编辑内容包括：

- 总金额。
- 分类。
- 备注。
- 标签。
- 退款。

点击“退款”可以输入退款金额，也可以点击“全额退款”。有退款的账单，统计和预算会按实际金额计算。

长按账单会进入多选删除模式。被长按的账单会默认选中，之后可以点击或滑动选择多条记录。底部会显示“已选几条”、删除和取消按钮。

删除是软删除，记录会进入“最近删除”，不是立刻永久删除。

## 四、流水页

流水页用于查看和搜索全部账单。

### 1. 查看流水

流水页按日期分组展示账单，展示规则与首页一致。首页和流水页的数据会同步：一边新增、删除或恢复，另一边也会更新。

### 2. 搜索账单

流水页顶部可以搜索账单。支持普通关键词，也支持带条件的自然语言。

示例：

```text
牛肉面
本月餐饮
昨天支出
超过100的购物
上月交通
```

搜索会尽量匹配账单标题、备注、分类、标签、时间和金额条件。

### 3. 删除与恢复

流水页也支持长按进入多选删除。删除后的记录进入“最近删除”。

## 五、统计页

统计页包含月报、周报和年报。

### 1. 月报

月报会展示当月收入、支出、结余、日均消费等数据。

月报里有一个小日历：

- 第一行显示周一到周日。
- 每个日期下方显示当天消费。
- 每行左侧显示该周消费。
- 点击某一天，会弹出窗口显示该日期的账单简记。

下方可以选择年份和月份。年份点击后展开选择，月份会显示成多行，当前选中的月份会有特殊样式。

### 2. 周报

周报按当前周统计收入、支出、结余、分类占比和趋势。

### 3. 年报

年报按当前年统计总收入、总支出、结余和分类结构。

### 4. 图表与排行榜

统计页包含：

- 每日趋势图。
- 分类占比图。
- 消费排行榜。
- 异常提醒。

退款账单会按实际金额参与统计，也就是总金额减去退款金额。

## 六、预算页

预算页用于设置和查看本月预算。

### 1. 添加本月总预算

输入预算金额后保存。预算默认按当前自然月生效。

### 2. 查看预算进度

预算进度会显示：

- 当前已用金额。
- 预算总额。
- 使用百分比。
- 状态：正常、接近上限、已超支。

退款后的账单会按实际金额计入预算。

### 2. 应用名称

可以自定义应用内显示名称。这个名称主要影响软件内部顶部标题，不一定能修改系统桌面图标下方的应用名。

### 3. 导出 CSV

点击“导出 CSV”会导出当前账单数据。

CSV 字段包括：

- 消费时间。
- 创建时间。
- 类型。
- 金额。
- 退款。
- 实际金额。
- 分类。
- 备注。
- 标签。
- 账户。
- 来源。
- 原始输入。

导出的文件会保存到应用外部文件目录的 `exports` 文件夹中。

### 4. 导入数据

点击“导入数据”会出现多种导入方式。

#### 导入 CSV（导出格式）

用于导入本软件导出的 CSV。适合从旧手机或备份文件恢复。

#### 导入 JSON（备份格式）

用于导入本软件备份 JSON。

#### 文字导入

点击后直接弹出文本框。把一段账单文字粘贴进去，程序会把文字交给“小票整理模型”生成结构化账单。

适合导入聊天记录、旅行消费清单、手写整理的消费明细等。

#### CSV / XLSX 转文字导入

选择 CSV 或 XLSX 文件后，程序会先在手机本机把表格内容转成纯文本。

CSV 转文本方式：

- 先解析逗号、换行和引号。
- 每一行保留为一行。
- 每一列用 Tab 分隔。

XLSX 转文本方式：

- 程序把 XLSX 当作 zip 文件读取。
- 提取 `xl/sharedStrings.xml` 和 `xl/worksheets/` 中的表格 XML。
- 解析单元格文本。
- 单元格用 Tab 分隔，行用换行分隔。

转换后的纯文本再交给“小票整理模型”生成账单。

#### 图片导入（小票图片识别）

选择图片后，程序会使用小票图片识别流程：

1. 图片交给百度 OCR。
2. OCR 返回纯文本。
3. 纯文本交给小票整理模型。
4. 生成账单并保存。

### 5. 大模型核对账单

大模型核对账单用于检查已有账单分类是否合理。

可以选择：

- 核对上次核对后新增的账单。
- 按日期范围核对。

点击后会先显示待核对账单，确认后再发送给小票整理模型。核对过程中会显示进度。

模型返回建议后，程序会弹出审核窗口，显示每条建议的修改前和修改后内容。你可以逐条选择是否接受。

接受建议后：

- 程序会修改对应账单。
- 如果出现新的大类、小类、消费项，经你确认后会加入识别规则。
- 后续相似输入会更容易被识别正确。

### 6. 打开导出文件夹

可以查看已经导出的 CSV、TXT 或 JSON 文件，并尝试打开或分享。

如果系统文件管理器无法直接进入 App 专属目录，可以在这个界面中直接打开或分享文件。

### 7. 手动备份 JSON

点击后会生成当前数据的 JSON 备份文件。建议在卸载应用、换手机、安装不同签名版本前先备份。

### 8. 最近删除

最近删除相当于回收站。

可执行操作：

- 单条选择恢复。
- 单条永久删除。
- 多选恢复。
- 多选永久删除。
- 清空回收站。

回收站会自动清理一个月前删除的记录。

## 八、识别规则

识别规则用于帮助程序把文字识别成正确的大类、小类和消费项。

### 1. 大类、小类、消费项

软件使用三层结构：

- 大类：餐饮、交通、购物、收入等。
- 小类：午饭、早餐、公交、地铁等。
- 消费项：牛肉面、咖啡、公交车、酒店等。

识别时，只要能匹配到大类、小类或消费项中的任意一个，程序就会尽量归类。只有完全无法识别时，才会走“其他”兜底。

### 2. 添加大类

在“识别词库”中可以添加大类。支持一次输入多个名称，用逗号、顿号或换行分隔。

### 3. 添加小类

选择一个大类后，可以添加小类。小类也支持一次添加多个。

例如：

```text
午饭，晚饭，夜宵
```

### 4. 添加消费项

可以选择大类，也可以选择该大类下的小类，然后添加消费项。

消费项同样支持批量输入：

```text
牛肉面，煲仔饭，麻辣烫
```

添加后会立即同步到识别规则中。手动添加的消费项优先级较高，会优先参与识别。

### 5. 删除规则

可以按大类、小类、消费项层级删除。

删除限制：

- 只有小类下消费项数量为 0 时，才能删除小类。
- 只有大类下小类数量为 0 时，才能删除大类。
- 删除消费项时，小类可选；如果选了小类，只能删除该小类下的消费项。

### 6. 查看所有规则

点击“查看所有规则”进入规则总览页面。

规则总览页面会显示所有大类。点击大类后，下方显示该大类下所有小类和消费项。

在这里可以：

- 直接编辑消费项文本。
- 删除消费项。
- 添加消费项。
- 长按小类删除小类。
- 通过“确认修改”统一保存。

修改保存前会弹出修改内容确认，确认后才生效。

### 7. 结构化批量添加规则

查看所有规则页面底部支持结构化批量添加。

格式示例：

```text
（餐饮），（午饭），（牛肉面，盖饭，炒饭）
（交通），（公交），（公交车，巴士）
（购物），（），（纸巾，洗衣液）
```

规则：

- 大类必填。
- 小类可填可不填。
- 消费项可填可不填。
- 多个消费项用逗号隔开。

### 8. 强约束规则

强约束规则用于处理容易混淆但方向明确的词。

例如：

- “吃”通常强绑定餐饮。
- “喝”通常强绑定餐饮。
- “高铁”强绑定交通。

强约束规则和平普通消费项规则隔离，可以单独添加和删除。它用于解决“面”既可能是牛肉面，也可能是面膜这类歧义。

## 九、模型配置

首页右上角“模型配置”可以进入模型部署与选择界面。

### 1. 文字记账模型

文字记账模型有三种通道：

- 本地规则：只走本地规则识别。
- 本地模型：使用手机本地模型。
- 云模型：使用云端文字模型。

只有点击右上角保存后，配置才会生效。如果只是切换按钮但不保存，下次进入仍显示当前已生效的通道。

### 2. API Key 隐私显示

API Key 默认以黑点隐藏。点击右侧眼睛图标可以显示或再次隐藏。每次重新进入模型配置页，API Key 默认都是隐藏状态。

### 3. 本地模型

可以导入本地文字模型。导入后程序会尝试初始化本地模型。是否可用取决于手机性能、模型格式和运行环境。

### 4. 云端文字模型

需要填写：

- Base URL。
- API Key。
- 模型名称。

云端文字模型用于首页文字记账的“云模型”通道。

### 5. 小票图片识别

需要填写：

- 百度 OCR API Key。
- 百度 OCR Secret / Service Key。
- 百度 OCR Endpoint。

图片识别会先调用百度 OCR，把图片转成纯文本。

### 6. 小票整理模型

需要填写：

- Base URL。
- API Key。
- 模型名称。

小票整理模型用于：

- 小票图片识别后的结构化整理。
- 文字导入。
- CSV / XLSX 转文字导入。
- 图片导入。
- 大模型核对账单。

## 十、桌面组件

软件包含桌面快记组件。

添加到桌面后，点击组件会打开一个小窗口。输入账单信息后点击“录入”，程序会自动识别并保存，成功后自动回到桌面。点击“取消”也会回到桌面。

## 十一、数据安全与备份建议

### 1. 本地数据库

账单默认保存在手机本机数据库。换手机、卸载应用、清除应用数据都会影响本地账单。

### 2. 定期导出

建议定期使用：

- 导出 CSV。
- 手动备份 JSON。

CSV 适合查看和迁移，JSON 更适合完整备份。

### 3. 云端模型风险

应用默认不联网。本地规则记账、手动记账、流水、统计、预算、规则管理等基础功能均可离线使用。只有当你主动配置并使用百度 OCR、DeepSeek 或其他云端模型时，软件才会向你配置的服务地址发送相关文本、表格内容或图片识别内容。请不要上传不希望第三方服务处理的敏感信息。

## 十二、常见问题

### 1. 为什么安装新版提示不能覆盖？

通常是签名不同。debug 版、未签名版和正式 CHENLH 签名版不能互相直接覆盖。需要先卸载旧版，再安装新版。卸载前请先备份。

### 2. 为什么输入一句话识别错了？

可能是规则库没有对应消费项，或关键词存在歧义。可以到“我的 - 识别词库”添加消费项、小类或强约束规则。

### 3. 为什么图片导入提示需要配置模型？

图片导入需要两部分配置：百度 OCR 和小票整理模型。两者缺一不可。

### 4. 为什么文字导入也需要小票整理模型？

文字导入、CSV/XLSX 转文字导入和图片导入最终都会把纯文本交给“小票整理模型”，由它生成结构化账单。

### 5. 退款后统计为什么变少？

这是正常行为。账单总金额用于展示，实际金额等于总金额减退款金额。统计、预算和搜索金额会优先使用实际金额。

### 6. 最近删除会一直保留吗？

不会。最近删除会自动清理一个月前删除的记录。

### 7. 软件是否收费？

当前版本用于个人自用和免费分享。云端模型、OCR 或第三方服务可能由对应服务商收费，费用与可用性由用户自行确认。
